# 第十四章：你的组织战略

> 原文：Chapter 14 · pp.129-144
> 阅读日期：2026-06-13

## 核心总结

商业发展计划七步中的 **第三步：组织战略（Organizational Strategy）**。

第十二章首要目标、第十三章战略目标定好之后，本章回答：**企业按什么结构运转，谁对什么结果负责**。

Levitt：组织若没有结构，就是 **暴民**——暴民搞建设不行，只会搞破坏。

### 反例：按「人」不按「岗」——Widget Makers 的灾难

多数公司 **按人格组织**，不按 **职能 / 责任** 组织 → 几乎总是混乱。

Jack 与 Murray 兄弟创业 **Widget Makers**：

1. **初期**：轮流干活（周一 Murray 开门，周二 Jack……），店很亮、顾客满意。
2. **雇侄子 Jerry、内兄 Herb**：仍是 **人人轮流、人人什么都干**——记账、接待、做 widget、买三明治、存钱……
3. **崩溃**：widget 退货、账目不对、工具丢失、车间脏乱、螺丝钉混装、互相撞肘、没人扫地擦窗、脾气变差。
4. **核心问题**：人人都干一切 → **谁对任何事负责？** 两个合伙人谁说了算？Jack 叫 Jerry 做事 Murray 不准怎么办？

> 没有组织架构图，一切靠 **运气、好感、人格**——单靠这些，是 **混乱与灾难** 的配方。

### 正例：重来一遍 Widget Makers, Inc.

Jack 与 Murray **在开业前** 换思路：

| 步骤 | 做什么 |
|------|--------|
| 1 | 当 **公司（corporation）**，不当「合伙人格」；对外 **股东（shareholders）**，对内 **员工（employees）** |
| 2 | 各写 **首要目标** |
| 3 | 完成 **战略目标**（Murray 做人口统计 / 需求调研，Jack 做财务预测） |
| 4 | 画 **组织架构图（Organization Chart）** |

**架构岗位（完成态）：**

- **首席运营官（COO / 总裁）** → 对战略目标总负责，向股东汇报
- **副总裁 / 营销** — 找客户、以更低成本更易方式满足 widget 带来的满足
- **副总裁 / 运营** — 按营销承诺交付、改进组装效率
- **副总裁 / 财务** — 盈利标准、融资
- 其下：销售经理、广告/调研经理；生产、服务、设施经理；应收、应付经理

两人意识到：**所有格子都要填名字——目前只有他们两个人。** 但图描述的是 **未来满员时的工作**，更重要的是 **现在就要按这些岗位干活**。

**岗位契约（Position Contract）：**

- **不是** 职位说明书，是公司与员工的 **契约**
- 含：岗位要产出的 **结果**、 **负责的工作**、 **评估标准**、 **签字**
- **问责（accountability）** = 「站起来，接受清点」——谁站起来、被清点什么

**分工：** Murray 任 COO？经斟酌 **Jack 当 COO**（更细致）；Murray 任 **营销副总裁** + 销售/广告调研经理等；Jack 兼运营、财务及多个一线经理。最初 Jack 8 岗 Murray 3 岗 → 调整为 **各 6 岗**。

**尚未开业一行活没干**，但已 conceive 公司、工作划分、标准、汇报关系—— **秩序感与可行感** 随之而来。组织架构图同时是 **特许经营原型的蓝图**。

### 从底层原型化：战术工作 vs 战略工作

原型化 **从组织最底层开始**，不是从顶层。

| | 战术工作（Tactical Work） | 战略工作（Strategic Work） |
|---|---------------------------|----------------------------|
| 谁做 | 技工做的 | 经理做的 |
| 目标 | 把活干完 | 设计、改进岗位 |

Jack 与 Murray **以员工身份** 在底层岗位干活，同时 **以副总裁身份** 对该岗位做 **商业发展流程（创新 / 量化 / 编排）**——每个岗位本身也是一个小 **特许经营原型**。

**Murray 做销售岗的例子：**

- 测服装颜色款式、测话术、测客户互动各环节 → 量化 → 写入 **销售操作手册**
- 手册含：来电/去电脚本、接待、询价/投诉回应、下单、退货、要货、库存……
- **手册完成之后才登招聘广告**——招 **新手 / 学徒**，不招「销售老手 / 大师傅技工」
- 广告：*COME AND SEE OUR TURN-KEY OPERATION… NO EXPERIENCE NECESSARY. JUST AN OPEN MIND…*
- 面试时展示手册、战略目标、组织架构、岗位汇报关系；谈对方 **首要目标** 是否契合
- 雇到人后：背话术、按着装规范、学系统 → **上岗瞬间 Murray 升为销售经理**，重复创新/量化/编排

Jack 对生产、应收应付等岗位 **同样处理** → 用 **系统** 替代自己在战术岗位的 presence，自己腾出来做战略工作。

**因果链（易混点）：**

```
首要目标 → 战略目标 → 组织架构图
（人生）    （企业标准）  （谁负责什么）
```

每一层是上一层的 **原因**；缺架构图 → 混乱、冲突；有架构图 → 方向、目的、风格 **平衡、有意图地** 推进。

### Sarah 的整合

- 画 **七年后完成态** 的组织架构图（不是现状）
- **把自己的名字填进** 目前承担的所有岗位
- 为每岗写 **岗位契约并签字**——「因为在你请人替代之前， **你就是** 每个员工」
- **必须像要求员工那样工作**，否则系统只会纵容老板偏好，无法让 **非老板** 的人高效、快乐
- 否则只能找「另一个老板型的人」来替代，永远离不开你

Gerber：**车主 / 司机（Gurdjieff）** 要驾驭马车——区分角色、排优先级、 **按你设计的规则亲自遵守**；领导不守则，没人跟。 **言行一致（walk your talk）**，企业才成气候。

章末 → **管理发展、人员发展**（第十五、十六章）。

## 案例卡片

### Widget Makers 轮流灾难

- **人物/场景**：Jack、Murray、Jerry、Herb 四人小厂
- **发生了什么**：人人轮流一切 → 质量、账目、整洁崩溃；无人问责
- **说明什么**：按人格不按岗位 = 混乱
- **关联概念**：第四章管理甩手；第九章最低技能 + 系统

### Jack & Murray 重来（Inc.）

- **人物/场景**：兄弟在厨房规划
- **发生了什么**：股东/员工分离 → 首要目标 → 战略目标 → 架构图 → 岗位契约 → Jack 任 COO
- **说明什么**：开业前结构先行；架构图 = 原型蓝图
- **关联概念**：第十一章七步；第八章原型

### Murray 销售手册再招人

- **人物/场景**：Murray 兼销售与营销副总裁
- **发生了什么**：创新/量化/编排 → 销售操作手册 → 招学徒 → 自己升销售经理
- **说明什么**：底层岗位原型化； **系统替代自己**
- **关联概念**：第十章 I-Q-O；第八章交钥匙

### Sarah 填格签字

- **人物/场景**：Sarah 与 Gerber 对话
- **发生了什么**：七年后架构图、名字进所有岗、签岗位契约；须像员工一样工作
- **说明什么**：否则系统纵容老板偏好，无法交钥匙
- **关联概念**：第十三章 7 年四家店；三人格
