# 引言（Introduction）

> 原文：Introduction · pp.2-6
> 阅读日期：2026-06-14

## 核心总结

全书讲 **如何与客户对话、从对话中学到真东西** —— 不是客户开发（Customer Development）全书，而是其中 **「怎么聊」** 这一块的实操手册。

### 核心隐喻：考古现场

从客户对话里学东西，像 **发掘脆弱的考古遗址**：

- 真相在地下，但 **一铲子太猛就碎**
- 有人用 **推土机 + 炸药** —— 逼人说好话（如「你觉得这主意好吗？」）
- 有人用 **牙刷** —— 不敢挖深，不知道底下有没有宝

> 用中文说清楚：  
> **真相是目标，问题是工具** —— 必须学会 **运用**；每个问题都可能 **带偏对方**，让整场对话白费。值得学，地下有宝藏。

### 这本书给谁

| 你是谁 | 处境 |
|--------|------|
| 读过精益创业 / 客户开发 | 不知如何 **开始第一次** 客户对话 |
| 传统销售 / 商务 | 想在还没商业模式的创业公司里更有效 |
| 导师 / 投资 / 支持方 | 想帮创始人聊得更有用 |
| 有了新点子 | 想验证是否值得 **辞职全力投入** |
| 在融资 | 投资人要 **真问题** 的证据 |
| 觉得整个过程 **尴尬** | 想要更轻的路 |
| 模糊感到机会 | 想弄清 **到底是什么** |
| 一直想创业 | 想 **今天就开始** 有进展 |

### 与客户对话很难

我们都知道 **该聊客户**，很多人也确实在聊——却仍做出 **没人买的东西**。聊客户本该防止这个，对吧？

**几乎所有人都聊错了。** 作者第一家公司 Habit（社交广告技术）：

- 三年拼命、钱快烧完、联合创始人被遣返、全职聊客户数月
- 最后发现：**聊法错了，等于白聊**

「要多和客户聊」像优等生对书呆子说「 **酷一点就好**」—— **没说有多难**。

| 坏对话的后果 | 说明 |
|--------------|------|
| **无用** | 浪费时间 |
| **更糟：假阳性** | 让你以为路对了 → 过度投入钱、时间、团队 |
| **客户总在撒谎** | 即便你没主动搞砸，他们也会误导你 |

本书是 **实操 how-to**，工具来自客户开发、设计思维、精益创业、用户体验、传统销售等；风格 **随意、去尴尬**，重点是 **会问好问题**。

### 为什么还要一本「聊 / 卖」的书

1. **作者是技术人，不是天生销售** — 内向、会议里笨拙；其他销售书写给 **已经会和人打交道的人**。他从零摸索，吃过「别打电话给我，我会打给你」的亏，后来也签下 Sony、MTV—— **课本和支票本之间有很大鸿沟**。
2. **先弄清怎么错，才能改对** — 全书会帮你找 **没意识到的聊法错误**。
3. **实用手册，非理论大部头** — 找谁聊、怎么约、怎么记笔记还不失礼，都写。

作者不教你怎么把公司做巨大——那是你的事；给的是 **对话工具、穿越噪音、弄清客户真正要什么**。创业最惨的是消失时 **没人在意**；本书要避免这个。

### 范围与术语

- **不是** 客户开发（Customer Development）的综述——那是 Steve Blank《四步创业法》《创业者手册》的大概念。
- **本书只管**：如何 **正确与客户对话并学习**。
- 为区分：日常聊天叫 **customer conversation（客户对话）**；大流程仍叫 **Customer Development（客户开发）**。
- 默认你已认同「该聊客户」——本书偏 **how**，不是 **why**。
