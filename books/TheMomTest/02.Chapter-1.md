# 第一章：妈妈测试（The Mom Test）

> 原文：Chapter 1 · pp.7-22
> 阅读日期：2026-06-14

## 核心总结

全书 **核心方法论** 登场：**妈妈测试（The Mom Test）** —— 一套问法规则，让连妈妈都没法对你撒谎。

### 别问「这主意好吗」

常说别问妈妈生意好不好—— **技术上对，但抓错重点**：

- **任何人都别** 用这话问——坏问题会 **邀请所有人至少撒一点谎**
- 妈妈撒最多（因为爱），但 **不是你的责任让别人说真话，是你的责任把真相挖出来**
- 靠 **好问题** 挖

### 失败版对话：iPad 数字食谱 App

儿子不断 **暴露自我、推销假想产品** → 妈妈：

- 礼貌性「嗯……」
- 在 **非真实购买情境** 里合理化 $40 价格
- 非承诺式恭维 + 功能建议（「会有图片吗？」）
- 最后请吃千层面（担心你 soon 吃不起饭）

儿子当成 **验证** → 辞职、砸积蓄 → **没人买（包括妈妈）**。

> 用中文说清楚：  
> **聊错比不聊更糟** —— 不知道时你还会小心；攥一把 **假阳性** 像劝醉汉他清醒了， **没有改善**。

### 通过版对话：同一产品，不同问法

| 差问法 | 好问法 | 学到什么 |
|--------|--------|----------|
| 「你会买食谱 App 吗？」 | 「iPad 上 **上次** 做了什么？」 | 旅行订房、娱乐+实用 |
| 推销功能 | 「订房用 App 了吗？」 | 妈妈用 Google；年轻人用 App Store 当搜索引擎 → **获客渠道** 可能不在商店 |
| — | 「其他 App 从哪知道的？」 | 周日报纸 App 专栏 → **传统 PR** 可行 |
| — | 「书架上新食谱哪来的？」 | 圣诞礼物、Marcy 送的、没拆封 → **老人不需要又一本文食谱**；**礼品市场**；**年轻厨师** 可能是更好细分 |
| — | 「你 **自己上次** 买食谱是什么时候？」 | 三个月前买素食食谱（爸要健康饮食）→ 老手仍买 **细分/小众** 食谱 |

**没提你的点子**，却得到：App Store 坐等不可行、两个客户细分、若干待验证风险。

### 有用对话的标准

早期客户对话 **是否有用**，看是否给出关于客户 **生活与世界观** 的 **具体事实** —— 才能改进生意。

| 模糊前提 | 聊完后的高保真 |
|----------|----------------|
| 老人喜欢食谱 + iPad → 做 iPad 食谱 | **细分1**：老手要小众食谱，难在 **不靠搜索触达**（报纸/杂志 PR 线索） |
| | **细分2**：年轻厨师，App Store 可达，但未验证是否愿为贵食谱付费 |

**第一次对话** = 给自己上吊的绳子；**第二次** = 可行动洞察。

**差别：** 第二次 **从头到尾没谈你的想法** —— 谈 **他们自己的生活**。

> 最终还是要介绍产品、收钱；但 **大错几乎总是提得太早，而非太晚**。  
> **不提想法** → 自动会问得更好 —— 这是客户对话 **最容易、收益最大** 的改进。

### 妈妈测试三规则

1. **谈他们的生活，不谈你的想法**
2. **问过去具体事实，不问泛泛或对未来的看法**
3. **少说多听**

名字来源：这样问，连妈妈都骗不了你；做对了，对方甚至 **不知道你有想法**。

**经验法则：** 客户对话 **默认是坏的** —— **你的工作是修好它**。

### 好问题 / 坏问题（闯关游戏）

| 问题 | 判定 | 要点 / 改法 |
|------|------|-------------|
| 「你觉得这主意好吗？」 | **烂** | 只有市场能判断；其余是意见 → 让对方 **演示现在怎么做**、成本、是否在找替代 |
| 「你会买做 X 的产品吗？」 | **坏** | 假想 + 乐观 → 几乎总 Yes → 问 **现在怎么解决 X**、上次发生何时、为何没解决 |
| 「你愿意为 X 付多少？」 | **坏** | 数字显得严谨仍是谎 → 问问题 **现在花多少钱**、预算多大 |
| 「梦想产品做什么？」 | **勉强** | 像排球 **垫球**，须追问 **为何** 要这些功能；收集动机与约束，不是功能清单 |
| 「你何必多此一举？」 | **好** | 从感知问题到真问题（例：要更好消息工具 → 其实要 **版本一致** → 解法像 Dropbox） |
| 「那有什么后果？」 | **好** | 区分 **愿付钱解决** vs **烦但能忍**；也是定价信号 |
| 「讲讲上次发生那次」 | **好** | **展示而非告知**；看行动；顺带摸清工作流、工具、约束、集成点 |
| 「讲讲你的工作流」 | **好** | （同上类） |
| 「还试过什么别的？」 | **好** | 现用方案、爱恨、切换成本；若抽象说「肯定付钱」却从未 Google → **不在乎** |
| 「付 X 买做 Y 的产品吗？」 | **坏** | 加数字也没用 → 问现状；够远的话 **直接要钱**（定金/预购） |
| 「你现在怎么应付？」 | **好** | **价格锚点**（£100/月 workaround vs £12万 agency） |
| 「钱从哪来？」 | **好** | B2B 必问；预算归属、谁能否决 |
| 「我还该跟谁聊？」 | **好** | **每场结尾**；不愿介绍 = 会开得糟或对方不在乎 |
| 「我还该问什么？」 | **好** | 给对方「纠正」你问线的借口；熟了可少用 |

**常用经验法则汇总：**

- 意见 **毫无价值**
- 涉及未来 = **过度乐观谎言**
- 以为你想听什么就 **说什么**
- 人知 **问题**，不知 **解法**
- 不懂目标 = **盲射**
- 有些问题 **其实不重要**
- 没主动找过解法 = **不会买你的**
- 人想帮你，但要 **你给借口**

### 用妈妈测试：谁定做什么

好问题 **都不是**「我该做什么产品」——

- 不是把创意交给 **委员会**
- **做什么由你定**；问的是客户的生活：问题、在乎什么、约束、目标
- 你谦卑收集信息 → **自己** 做愿景跳跃 → 用 **承诺与推进（Ch5）** 验证细化

> **他们不替你定义问题，你不让他们定义方案。**  
> **问题归他们，方案归你。**

## 案例卡片

### iPad 食谱：失败 vs 通过

- **人物/场景**：与妈妈的两次对话

### 财务小哥要消息工具 → Dropbox

- **人物/场景**：「你何必多此一举」追问
- **发生了什么**：表面要聊天工具，真要 **最新版表格**
- **说明什么**： **动机 > 功能请求**
- **关联概念**：梦想产品追问

### 「肯定付钱」却从未 Google

- **人物/场景**：「还试过什么」深挖
- **发生了什么**：抽象热情 vs 零搜索行为
- **说明什么**： **抽象情绪 ≠ 真实需求**
- **关联概念**：鞋带双结比喻

### 灾难工作流 vs 实习生搞定

- **人物/场景**：「有什么后果」
- **发生了什么**：大喊 DISASTER，实际扔实习生就挺好
- **说明什么**： **问题存在 ≠ 值得付费解决**
- **关联概念**：E-Myth 感知需求
