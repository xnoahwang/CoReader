# 第二章：避开坏数据（Avoiding Bad Data）

> 原文：Chapter 2 · pp.23-39
> 阅读日期：2026-06-14

## 核心总结

第一章给了 **妈妈测试**；本章讲 **对话仍会跑偏时怎么办** —— 识别并处理 **三种坏数据**。

> 用中文说清楚：  
> 给傻瓜信息会让他破产。坏数据带来 **假阴性**（以为点子死了其实没有）和更危险的 **假阳性**（以为对了其实错）。  
> 就算努力遵守妈妈测试，也会因兴奋开 pitch、为解释会议不得不谈想法、或陷在假想未来里而偏离。

### 三种坏数据

| 类型 | 中文 | 典型形态 |
|------|------|----------|
| **1. 恭维（Compliments）** | 「好酷」「喜欢」 | 会议结束几乎总有 |
| **2. 空话（Fluff）** | 泛泛、假想、未来 | 「通常」「我会」「可能会」 |
| **3. 点子（Ideas）** | 对方给你的功能建议 | 兴奋时翻到你这边一起脑暴 |

**三种应对：** **挡开恭维 · 锚定空话 · 挖透点子**

---

### 挡开恭维（Deflect compliments）

多数会议以恭维收场—— **感觉好，但几乎一定在撒谎**（未必故意：想支持、护你感受、被你兴奋传染）。

即便真喜欢，**数据仍无价值**—— VC 专业判断都常错，路人观点能有多重？

**要事实与承诺，不要恭维。** 最好 **完全不提想法**；恭维来了就 **挡开**，继续挖事实。

#### 坏对话 vs 好对话（pitch 后）

| 坏 | 好 |
|----|-----|
| 「酷，喜欢」→「谢谢！会议很成功！」 | 「酷」→ **道歉滑入 pitch** →「你们这行怎么 **现在** 处理这事？」 |
| 办公室：「人人都爱！」→ 六月后零客户 | 挖到：两人全职 + Excel + 邮件 → **问题真值钱** |
| | 或 **扎实否定**：「其实不大回事，我们忽略」→ 也是好对话 |

**好对话不必听到想听的** —— 只要 **到真相**。

恭维 **无处不在**，总在骗你以为「聊得不错」。

**症状（会上）：**「谢谢！」「很高兴你喜欢！」  
**症状（会后）：**「反馈很正面」「人人都爱这主意」→ **全是警报**

> **恭维是客户调研的 fool's gold**：闪亮、分心、 **完全无价值**。

---

### 锚定空话（Anchor fluff）

空话三形：

| 形 | 例子 |
|----|------|
| **泛泛宣称** | 「我通常 / 总是 / 从不…」 |
| **未来承诺** | 「我会 / 我愿意…」 |
| **假想可能** | 「我可能会 / 我可以…」 |

**最致命空话：**「 **我肯定会买**。」—— 听起来具体，创始人最想信；但人对 **想象未来** 总比 **现实到来时** 更乐观、更愿付钱。

作者第一家创业公司：**把空话当承诺** → 亏约 **1000 万美元**。

**诱发空话的问题：**「你会…吗？」「你通常…？」「你觉得你会…？」「你能想象自己…？」  
—— 不是绝对不能问，而是 **答案没用**；错在 **当真**，不在提问。有时可 **过渡到具体问**：

```
你：「你会遇到 X 吗？」→ 对方：「经常啊」（空话）
你：「上次什么时候？」→「两周前」→ 锚住了
你：「能讲讲那次怎么回事吗？」→ 回到好问题
```

#### 收件箱案例

「我是 Inbox 0 狂热者」→ **「你现在收件箱多少？」** → 10 封；**「上次彻底崩了何时？」** → 旅行断网 10 天才恢复 → **锚到真实边缘情况**

泛泛时人描述 **想成为的自己**，不是 **真实的自己**。

#### 忠诚卡 App 案例

| 坏 | 好 |
|----|-----|
| 听到抱怨 → pitch →「你会用吗？」→「肯定用！」 | 「试过手机 loyalty App 吗？」→「从没下载」→「现在下？」→「下次吧」= **不是真问题** |
| | 或：下过好几个、要折扣更多、嫌现有 App 难用 → **真信号** |

> 「总该有人做 X！」「你找过吗？」「没，其实也不需要。」—— **抱怨者 ≠ 客户**。

### 挖透点子（Dig beneath ideas）

创业者 **不缺点子**；对方兴奋时也会 **翻到你这边** 列一堆功能—— **记下，别急着进 todo**。

**错误：** 听到「要同步 Excel」→ 写上清单就走 → **功能蠕变快车道**

**正确：** 挖动机——

| 他们说的 | 可能意味 |
|----------|----------|
| 「同步 Excel 能干嘛？」→ 偶尔翻翻老报表，不是关键 | 别做 |
| 「试过好几家，都死在同步」 | **必配差异化** |
| 「每月末凑报告要一周，现有凑合但痛」 | **理想早期客户** |

#### Habit / MTV 教训

MTV 要 **分析报表** → 作者照单全做华丽 dashboard → 对方每周五仍要他 **发 CSV/PDF** → 才发现：

> 他们要的不是 **懂数据**，是 **让自家客户每周收到好看邮件**。

若早问清动机：固定导出几个高层数字 + 每周定时发 + 实习生排版即可—— **不必 dashboard**。 **三个月白费**。

**听到请求 → 挖根因：** 为何这样做事？为何要这功能？没有它现在怎么凑合？  
**强烈情绪也要挖：** 愤怒、尴尬、狂喜——「多说说」「这似乎很烦——背后有故事吧？」

> **点子与功能请求：要理解，不要盲从。**

**挖功能：** 为何想要？能让你做什么？没有它怎么应付？先上线还是后可加？怎么融入一天？  
**挖情绪：** 多说说；真很烦吧；什么这么糟；为何还没修好；这么兴奋——大事？

### 别求认可（Stop seeking approval）

恭维常来自 **求认可** —— 有意 **钓鱼**（「你觉得能成吗？」）或无意 **悲情问题（Pathos Problem）** —— 暴露自我，别人觉得 **该护着你**。

| 钓鱼症状 | 悲情症状 |
|----------|----------|
| 「我在想创业…能成吗？」 | 「我辞职做的秘密项目…说实话你觉得怎样？」 |
| 「App 主意超棒——你喜欢吗？」 | 「我能接受批评——请诚实！」 |

**解法：** 焦点放回 **对方** + **过去具体事例**。一旦察觉 **你的自尊在桌上**，就会收到空话与额外恭维—— **扔掉**，用妈妈测试拉回。

马斯克们主动求骂行得通—— **因为他们是马斯克**；你我须 **更谨慎**。

> 恭维无价值；认可 **不能让生意变好**。想法与自尊 **别进对话**，直到真要 **要承诺**。

---

### 切断推销（Cut off pitches）

推销是求认可的 **阴暗面**—— **不让你走直到你说喜欢**；恭维只是想把人请出办公室。

**症状：**「不不，你没懂…」「对，但它还能…」

滑入 pitch → **道歉**：「刚滑进推销模式——能跳回你刚才说的吗？」若对方坚持要听 → 承诺 **会后再讲 / 早期 demo**，先 **多聊他们的**。

> 你烦够久，谁都会说好。

### 少说（Talk less）

不闭嘴几分钟 **学不到有用东西**。介绍想法后他们想「所以这跟…类似」「我喜欢但…」—— **别打断纠正**；他们正要给你 **世界观窗口**。安全等重要话题也别抢着答「我们想过了」。

> **你说得越多，做得越差。**
