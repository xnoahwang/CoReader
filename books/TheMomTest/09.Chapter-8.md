# 第八章：运行流程（Running the Process）

> 原文：Chapter 8 · pp.98-110
> 阅读日期：2026-06-14

## 核心总结

对话技术都对， **流程不对仍会得到坏结果** —— 光开会碰运气是浪费所有人时间；会前会后要做一点工作 **榨干价值**。

全书实操收束：**会前准备 · 会后复盘 · 谁参会 · 怎么记 · 总流程 · 时间尺度**。

---

### 学习瓶颈（Learning bottleneck）

**反模式：** 商务 guy 独自见客户 → 回来告诉团队「该做什么」= **「我学到的」≈「你们要做的」** → 握客户对话者成 **事实独裁者**，「客户这么说的」是 **终极王牌**。

客户话 **极易误读**；学习卡一人脑子里 = **瓶颈**。

**最极端：** 独自开会 + 烂笔记 + 不复盘团队 → 你的头 = **客户真理唯一仓库**。

作者：**CTO 离职**——「你再改方向我们永远成不了」——学到的或许真，但 **没传达给团队**。

**瓶颈症状：**

- 「你管产品，我去学」
- 「因为客户告诉我的！」
- 「我没时间聊人——得写代码！」

**避免瓶颈三件套：** 准备（Prepping）· 复盘（Reviewing）· 好笔记

---

### 会前准备（Prepping）

**最重要：** 团队对齐当前 **三大问题**（Ch3），敢面对吓人问题。

若已摸清事实 → 也定好会末 **要什么承诺/下一步**。

**写信念骨架：** 花最多一小时猜对方在乎什么——多半错，但有骨架 **更好控场**；细分够窄则很少要做。

- 能 **桌面研究** 答的 → 先查，把对话留给 **互联网给不了** 的
- 见企业前 LinkedIn/官网 **5 分钟尽职调查**

**全创始团队一起 prep** —— 商务+产品都要在；漏一方 = 客户对话里 **漏他们的顾虑**。

「该做产品别瞎聊」的抵触 → 表面拒面对现实，实为 **把速度=堆功能** 而非 **去风险/验证**。

**挖隐藏风险两问（直觉即可）：**

1. **若公司会死，最可能死于什么？**（例：卡通社区——没人创作 / 作品太烂没人看）
2. **若要巨大成功，必须为真的是什么？**（Lafley/Martin 战略书翻转版）

→ 核心：**「我们想从这些人身上学到什么？」** 时间紧 **只问这个也够**。

> **不知要学啥，就别开会。**

---

### 会后复盘（Reviewing）

跟团队 **过笔记**，更新信念与三大问题。

- 关键引语、要点、踩坑
- **元层面：** 哪些问题有效？下次怎么更好？漏了什么信号？

看似简单 **别跳过** —— 用引语 **尽快、直接** 传播学习；同步、好决策、少吵架、全团队受益。

---

### 谁该到场

**做大决策的人（含技术）** 都应参加 **至少一些** 会——技术不必场场，但 **亲耳客户反应** 无价；互相揪对话错误与偏见。

**最佳两人：** 一说一记；第二人可 **打断修正** 烂问题或漏挖的信号。

**别超过两人**（除非群对群或有理由）—— 三人面试略压迫。

**单人** 可以，但要 **会记简笔记**；难自我纠错。

害羞无联合创始人 → 朋友先来几场你 **只记笔记**；信心 **分领域**，练就会不那么怕。

**客户学习不能外包** —— 雇人带来坏消息难消化；温吞信号雇人易 **粉饰**。在 **可重复销售/营销模型** 之前， **创始人必须在场**。

可雇人 **安排/主导** 会议，但 **你得在听**。

---

### 怎么记（How to write it down）

好笔记 = 团队/投资人/顾问同步 + **难自欺** + 数月后调方向 **不必重访**。

- **尽量逐字引语**，加引号
- 或记大意 + **符号速记**

**作者常用符号（可自建）：**

| 符号 | 含义 |
|------|------|
| :) :( :\| | 兴奋 / 愤怒 / 尴尬 |
| ☇ | 痛点 |
| ☐ | 障碍 |
| ⤴ | 变通办法 |
| ^ | 背景 |
| ☑ | 功能请求或采购标准 |
| ＄ | 钱/预算/采购流程 |
| ♀ | 提到具体人/公司 |
| ☆ | 待跟进 |

**情绪+生活** 组合：说「是个问题」中性 vs 暴怒 **完全不同**。

**障碍：** 想解决但被挡住（如公司 IT 政策禁云）—— 变通可能是私人手机、回家干活。

**存放要求：** 可排序、团队合并、持久可查、 **不与 todo 混**

- 主笔记本几个月后 **几乎没法检索** = 没用
- **Google 表格 / Evernote** 好分享搜索；会上电脑记略失礼 → 会后誊录
- **作者最爱索引卡 / 便利贴**：一卡一引语+符号；复盘 **摊桌排序**；痛点不成立可 **抽出所有闪电符卡** 换已验证问题
- 录音：人常同意，但难检索；且 **破坏随意聊**

**随意聊** 笔记可能在纸盘、杯垫、报纸上—— **先记下再迁回主系统**；pub/会中不便记 → 聊完 **躲角落立刻写**

> **笔记不回顾 = 无用。**

### 流程清单（全书汇总）

对话是 **工具非义务**；半心半意调查问卷 **不如不做**。

**走过场信号：**

- 你说得比对方多
- 对方恭维你或点子
- 说了想法不知下一步
- 没笔记 / 没跟团队过笔记
- 意外答案却 **没改想法**
- 没有怕的问题
- 不知这场要答 **哪个大问题**

**一批对话前：**

1. 选定 **聚焦、可找到** 的细分（Ch7）
2. 团队定 **三大学习目标**
3. 若适用：理想 **下一步与承诺**
4. 若对话是正确工具：找谁聊
5. 写对方在乎什么的 **最佳猜测**
6. 能桌面研究的先研究

**对话中：** framing（Ch6 VFPA）· 随意（Ch4）· 妈妈测试 · 挡恭维/锚空话/挖信号 · 好笔记 · 若适用推承诺

**一批对话后：**

1. 团队过笔记与引语
2. 迁入永久存储
3. 更新信念与计划
4. 定 **下一组三大问题**

**目标：** 时间花在 **真要紧的问题** + 学习 **全团队最快传播**。

结合：**妈妈测试 + 随意 + 推进** → 最少时间最大学习。

---

### 时间要快（This stuff is fast）

流程为 **加速生意**，不是拖慢。

- 别 **一周** prep—— **一小时** 就去聊；再多 = **拖延**
- 别 **全职聊几个月** 才动产品—— **一两周** 摸方位，然后给东西 **让他们承诺**
- 公司一生都要聊客户；用随意聊 **几乎不占额外时间**，与增长 **并行非替代**

> **Go build your dang company already.** —— 去把你该建的公司建起来。
