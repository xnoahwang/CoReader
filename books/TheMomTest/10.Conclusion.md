# 结论与速查表（Conclusion and Cheatsheet）

> 原文：Conclusion · pp.111-118
> 阅读日期：2026-06-14

## 核心总结

全书 **收尾 + 可打印速查表** —— 作者仍会犯蠢问、滑入 pitch； **注意到就有机会修**。

> **会搞砸没关系。** 跟团队复盘，别苛责自己或联合创始人。好对话大多 **能救回来**；你在做难事，不必完美，但要 **越来越好**。

大家 **支持创业者** —— 搞砸了也愿原谅。好问题能在麻烦前 **修正很多烂点子**；仍失败也 OK，去谢帮过你的人，他们会说「别在意，懂的」。

> **会好起来的（It's going to be okay）。**

### 砍绳结（Hack it）

亚历山大 **一剑砍断** 戈尔迪之结，而非按「正确流程」慢慢解。

客户细分工作坊：私教通勤耗非计费时间 → 有人建议警察客户（驻站、少通勤、出价不敏感）→ 众人 geek 验证计划时，他举起手机：**「我直接打给他们不行吗？」** —— 20 分钟后约到试课。

**有流程有价值，但别卡死在流程里** —— 有时 **拿起电话就砍过去**。

### 妈妈测试三规则

1. **谈他们的生活，不谈你的想法**
2. **问过去具体事实，不问泛泛或未来意见**
3. **少说多听**

### 回到正轨（避开坏数据）

- **挡开恭维**
- **锚定空话**
- **挖透意见、点子、功能请求、情绪**

### 常见错误与症状

| 类型 | 症状句 |
|------|--------|
| **钓鱼恭维** | 「我在想创业…能成吗？」「App 主意超棒——你喜欢吗？」 |
| **悲情问题（暴露自我）** | 「我辞职做的秘密项目…你觉得怎样？」「我能接受批评——请诚实！」 |
| **推销模式** | 「不不你没懂…」「对，但它还能…」 |
| **太正式** | 「感谢参加访谈…」「1–5 分你多大程度…」「我们约个会吧」 |
| **学习瓶颈** | 「你管产品我去学」「客户这么说的！」「没空聊人要写代码！」 |
| **收集恭维非事实** | 「反馈很正面」「人人都爱这主意」 |
| **走过场** | 你说得多、对方恭维、不知下一步、无笔记、未团队复盘、意外答案未改想法、不怕任何问题、不知本场答哪个大问题、不知为何开会 |
| **不推承诺** | 僵尸线索、以恭维收场、无下一步、「会开得不错」、对方没付出有价值的东西 |

### 一批对话流程

**会前：**

1. 选定 **聚焦、可找到** 的细分
2. 团队定 **三大学习目标**
3. 若适用：理想 **下一步与承诺**
4. 确定找谁聊
5. 写对方在乎什么的猜测
6. 能桌面研究的先做

**会中：** framing · 随意 · 妈妈测试 · 挡/锚/挖 · 好笔记 · 推承诺

**会后：** 团队过笔记与引语 · 迁入永久存储 · 更新信念与计划 · **定下一组三大问题**

### 好会的结果

| 产出 | 含义 |
|------|------|
| **事实** | 具体、关于他们 **做什么、为何做** —— 非恭维/空话/意见 |
| **承诺** | 付出时间、声誉或金钱 |
| **推进** | 漏斗 **下一步**，更接近成交 |

### 会前一句（最重要）

> **「我们想从这些人身上学到什么？」**

### VFPA 约见框架

| 要素 | 内容 |
|------|------|
| **Vision 愿景** | 半句话：你如何让世界更好 |
| **Framing 框定** | 阶段 + 在找什么 |
| **Weakness 示弱** | 你需要什么帮助 |
| **Pedestal 捧对方** | 为何 **尤其** 他们能帮 |
| **Ask 请求** | 请帮助 |

口诀：**Very Few Wizards Properly Ask**
